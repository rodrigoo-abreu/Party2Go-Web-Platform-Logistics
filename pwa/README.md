# Party2Go PWA

Vue 3 + Vite **Progressive Web App** for Party2Go with **mobile-first** design and Bootstrap 5.

## Tech Stack

- **Vue 3** with `<script setup>` syntax
- **Vite** for fast development and optimized builds
- **Bootstrap 5** (v5.3.8) for responsive UI components
- **Vue Router** v4 for client-side routing
- **Pinia** v2 for state management
- **Vite PWA Plugin** for offline support and service workers

## Features

- Mobile-first responsive design with Bootstrap 5
- Progressive Web App (PWA) with service workers for offline support
- Modular Vue 3 components with scoped CSS
- Component-based architecture (SignIn view composed of 4 sub-components)
- Mobile-optimized touch targets (56px minimum button heights)
- Vue Router integration with SignIn on root path (/)
- Vite path alias support (@/ → ./src)
- Global CSS variables for theming

## Project Setup

### Install dependencies

```bash

```

### Development Server


## Component Architecture

### SignIn View (Mobile-First)

The SignIn view is modularized into 4 focused sub-components with mobile optimization:

```
views/SignIn.vue (Composition parent)
├── components/SignInLogo.vue      (Logo display)
├── components/SignInHeader.vue    (Red header with icon)
├── components/SignInForm.vue      (Form fields - 56px touch targets)
└── components/SignInFooter.vue    (Login link)
```

**Features:**

- Full-width layout (col-12) on mobile
- 56px minimum button height for touch targets
- Loading state during form submission ("A registar...")
- Responsive padding and spacing
- Data Flow: Form submission via emits from child components to parent

### Styling

- Global styles in `src/assets/style.css`
- Component-scoped styles in each `.vue` file
- Bootstrap 5 utility classes (col-12, py-3, gap-\*, etc.)
- Mobile-first responsive breakpoints
- CSS variables for colors and typography

## PWA & Offline Support

- Service workers automatically generated by Vite PWA Plugin
- Offline functionality enabled
- App manifest for installability
- Cache strategies configured in `vite.config.js`
